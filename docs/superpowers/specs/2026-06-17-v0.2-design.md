## 一、结构变更：单 crate → workspace

```
motif/
├── Cargo.toml              # [workspace] + [package] 合一（主 crate）
├── src/                    # 不变
├── motif-macros/           # 新增：proc-macro crate
│   ├── Cargo.toml          # proc-macro = true, 依赖 syn/quote/proc-macro2
│   └── src/lib.rs          # #[tool] 宏实现
└── tests/                  # 不变
```

主 crate 依赖 `motif-macros = { path = "./motif-macros" }`，在 `lib.rs` 中 `pub use motif_macros::tool;`。

## 二、新增功能

### 2.1 LLMProvider::call_stream — 流式调用

```rust
#[async_trait]
pub trait LLMProvider: Send + Sync {
    // 非流式（已有）
    async fn call(&self, messages: &[Message], tools: &[ToolDefinition]) -> Result<LLMResponse>;

    // 流式（新增）
    async fn call_stream(
        &self,
        messages: &[Message],
        tools: &[ToolDefinition],
    ) -> Result<LLMStream>;
}

/// 流式响应包装
pub struct LLMStream {
    pub receiver: tokio::sync::mpsc::Receiver<StreamEvent>,
}

pub enum StreamEvent {
    Content(String),       // 文本增量
    ToolCallDelta(String), // 工具调用增量（JSON片段）
    Thinking(String),      // 推理过程增量
    Finish(FinishReason),  // 流结束
}
```

`OpenAIProvider` 使用 SSE 解析实现 `call_stream`。Agent 内部用 `call` 做逻辑判断，暴露流给外部hooks。

### 2.2 #[tool] 过程宏

```rust
// 使用方
use motif::tool;

/// 搜索互联网
#[tool]
async fn search(
    /// 搜索关键词
    query: String,
) -> String {
    format!("搜索结果: {}", query)
}

// 展开后等价于：
// - 生成 SearchArgs { query: String }
// - impl ToolArgs for SearchArgs { TOOL_NAME = "search", ... }
// - 原函数签名变为 async fn search(args: SearchArgs) -> String

let agent = Agent::new(provider)
    .tool(search);  // 直接传函数，宏已生成注册逻辑
```

### 2.3 bind() — 方法级工具

```rust
#[tool]
impl Database {
    /// 查询数据库
    async fn query(self, sql: String) -> String { ... }
}

let db = Database::new();
let agent = Agent::new(provider)
    .bind(db.clone(), Database::query);
```

### 2.4 FinishReason::Length 续写

LLM 返回 `Length` 且 content 非空时，追加 "continue" user message 让 LLM 继续输出。最多 3 次。

### 2.5 空响应重试

content 为空 + 无 tool_calls + finish_reason == Stop 时，自动重试最多 2 次。2 次后仍空 → 返回空。
